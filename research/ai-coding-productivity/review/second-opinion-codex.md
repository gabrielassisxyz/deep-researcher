# AI Coding Productivity Second Opinion

This is an independent Gate 8.4 second-opinion dossier. It did not read the primary run in `research/ai-coding-productivity/`.

## Main Finding

The evidence does **not** support a general claim that AI coding agents make experienced developers measurably faster on real work in codebases they already know.

The evidence supports a narrower claim: AI tools can speed up constrained, well-specified, implementation-heavy work, especially for less-experienced developers. For experienced developers in familiar mature codebases, the best direct RCT found slowdown, and the most current agentic-tool evidence is not yet clean enough to quantify.

## Highest-Impact Caveat

Self-reported productivity is not a substitute for measured productivity. The most direct RCT found a large perception gap: developers believed AI sped them up even when measured task time was slower (Source: [[sources/metr-early-2025-rct]]).
