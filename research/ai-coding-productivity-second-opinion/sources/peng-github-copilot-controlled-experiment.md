# Peng et al. GitHub Copilot Controlled Experiment

## Source Type

Controlled experiment released as an arXiv preprint and Microsoft Research publication. Vendor-affiliated because the study concerns GitHub Copilot and includes Microsoft/GitHub-linked researchers.

## Method

Recruited software developers were asked to implement an HTTP server in JavaScript as quickly as possible, with treatment access to GitHub Copilot.

## Key Evidence

- Treatment participants completed the task 55.8% faster than control participants.
- The studied task was short, standardized, and greenfield rather than work in an existing familiar codebase.
- The abstract says heterogeneous effects showed promise for helping people transition into software-development careers, implying benefits were especially relevant to less-experienced developers.

## Caveats

This is often cited as evidence that Copilot makes developers faster, but its direct external validity to experienced developers doing real work in familiar production codebases is limited. It supports speedup on a constrained implementation task, not the full user claim.

## Practical Implication

The study supports gains for well-scoped greenfield implementation. It does not settle experienced-developer productivity in familiar codebases.
