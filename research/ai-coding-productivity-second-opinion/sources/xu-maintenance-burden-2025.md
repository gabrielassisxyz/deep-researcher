# Xu et al. Maintenance Burden

## Source Type

arXiv preprint, presented at several 2025 conferences according to the arXiv record. Treat as independent working-paper evidence unless peer-review status is separately verified.

## Method

Observational analysis of open-source software project activity following GitHub Copilot introduction.

## Key Evidence

- The paper finds overall productivity increases after AI adoption.
- The increase is primarily driven by less-experienced or peripheral developers.
- Code written after AI adoption requires more rework to satisfy repository standards.
- More experienced core developers review 6.5% more code and show a 19% drop in original code productivity.

## Caveats

Observational design can be confounded. The abstract does not by itself settle causality or all measurement definitions.

## Practical Implication

Aggregate productivity gains can hide burden shifting onto experienced developers. This directly bears on the user's question because the experienced developer may become faster at some tasks but slower overall due to review and rework load.
