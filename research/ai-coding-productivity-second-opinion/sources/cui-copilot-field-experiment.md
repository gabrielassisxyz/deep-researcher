# Cui et al. Copilot Field Experiment

## Source Type

Field experiment report on MIT PubPub. It studies Microsoft and Accenture developers, and the tool is GitHub Copilot, so independence is limited by organizational involvement and data access.

## Method

Two field experiments with 1,974 software developers at Microsoft and Accenture. A random subset received GitHub Copilot access.

## Key Evidence

- Preliminary estimates suggest developers completed 12.92% to 21.83% more pull requests per week at Microsoft and 7.51% to 8.69% more at Accenture, depending on specification.
- The authors emphasize low compliance in the Microsoft experiment and internal organizational changes at Accenture.
- Estimates were not very precise and reached statistical significance only under weighting that emphasized periods with larger uptake differences.

## Caveats

Pull-request count is an output metric, not direct measured task time or value. The estimates are suggestive rather than decisive. Compliance and organizational changes reduce confidence.

## Practical Implication

This is evidence for possible real-world throughput gains in professional organizations, but not strong enough to override contradictory RCT and observational findings.
